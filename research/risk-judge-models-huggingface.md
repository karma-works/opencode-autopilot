# Risk Judge Models: HuggingFace Research

> Research conducted: 2026-05-25

## Executive Summary

No single model exists purpose-built for "bash command / file write / network request ALLOW/DENY" classification in a coding agent context. The most practical candidates, ordered by fit:

1. **sh-guard** — fast rule-based AST classifier (Rust, zero latency) — best deterministic pre-filter
2. **ShieldAgent (8B, MIT)** — takes full tool-call interaction history with tool definitions; structured output; directly mappable
3. **TS-Guard (8B, Apache 2.0)** — first open-weight model specifically trained for pre-execution tool invocation safety
4. **Llama Guard 3-1B (Apache 2.0)** — Ollama-compatible, S14 Code Interpreter Abuse category, custom categories injectable; best for low-latency local use
5. **Llama Prompt Guard 2-86M** — prompt injection detection for scanning tool outputs; complements (not replaces) the action judge

## Tier 1: Purpose-Built for Agent / Tool Call Safety

### 1. TS-Guard (ToolSafe)

- **HuggingFace:** https://huggingface.co/MurrayTom/TS-Guard
- **Size:** 8B (BF16)
- **License:** Apache 2.0
- **Specifically designed for:** Step-level tool invocation safety — reasons over interaction history to detect harmful tool invocations *before* execution
- **Input:** Full agent interaction history (user messages, tool calls with arguments, tool results)
- **Output:** Safe / Unsafe classification + interpretable reasoning chain
- **Training:** Multi-task RL on TS-Bench, a novel benchmark for step-level tool invocation safety
- **Benchmarks:** 65% reduction in harmful tool invocations on ReAct agents; ~10% improvement in benign task completion under prompt injection on TS-Bench
- **Local/Ollama:** Via Transformers (GPU required). llama.cpp/Ollama compatibility unconfirmed.
- **Caveats:** Very new (January 2026), low download count. Base model architecture not published.
- **Paper:** https://arxiv.org/abs/2601.10156

### 2. Agent Action Guard (MCP Action Classifier)

- **HuggingFace Blog:** https://huggingface.co/blog/prane-eth/agent-action-guard
- **GitHub:** https://github.com/Pro-GenAI/Agent-Action-Guard
- **Size:** ~22-66M (MiniLM embeddings)
- **License:** Not explicitly stated
- **Specifically designed for:** MCP-style action safety. Classifies MCP action objects as Safe / Harmful / Unethical
- **Input:** MCP action objects (tool name + arguments)
- **Output:** Three-way classification with risk level
- **Training:** HarmActions dataset (123 entries) — covers unsafe code execution, sensitive file modifications, risky API interactions, harmful messages
- **Benchmarks:** 90.32% accuracy on HarmActionsEval
- **Local/Ollama:** Yes — designed for edge deployment, no GPU required. Quantized variants available.
- **Caveats:** Training set is tiny (123 entries) — may not generalize to novel attack patterns. Three-way taxonomy doesn't map cleanly to ALLOW/DENY.
- **Paper:** https://www.preprints.org/manuscript/202510.1415

### 3. ShieldAgent (thu-coai)

- **HuggingFace:** https://huggingface.co/thu-coai/ShieldAgent
- **Size:** 8B (F16), base: Qwen-2.5-7B-Instruct
- **License:** MIT
- **Specifically designed for:** Agent behavioral safety judgment — evaluates whether tool invocations in agent interactions are safe
- **Input:** Structured prompt containing: safety evaluation instructions + background environment with tool definitions (name, description, arguments) + dialogue history including user messages, assistant tool calls (tool_call_id / tool_name / tool_arguments), and tool execution results
- **Output:** `[Answer] 'safe' or 'unsafe'\n[Analysis] Detailed explanation`
- **Training:** 4,000 agent interaction records with GPT-4o-generated labels, from Agent-SafetyBench
- **Benchmarks:** 91.5% accuracy vs GPT-4o baseline of 75.5% on 200-record test set (Gemini-1.5-Flash interactions)
- **Local/Ollama:** Yes, via Transformers. GGUF quantizations likely available given Qwen base.
- **Caveats:** Small test set (200 records). Training labels generated by GPT-4o — not adversarially hardened.
- **Paper:** https://arxiv.org/abs/2412.14470

## Tier 2: Prompt Injection Detection (Composable Layer)

These do not classify tool call risk but are valuable as Layer 3 — scanning tool outputs before they re-enter the agent's context.

### 4. Llama Prompt Guard 2 (Meta)

- **HuggingFace (86M):** https://huggingface.co/meta-llama/Llama-Prompt-Guard-2-86M
- **HuggingFace (22M):** https://huggingface.co/meta-llama/Llama-Prompt-Guard-2-22M
- **Architecture:** mDeBERTa-v3 base (BERT-style encoder — fast inference, not a generative model)
- **License:** Llama 4 Community License (commercial use permitted below 700M MAU)
- **Input:** Text prompt (up to 512 tokens)
- **Output:** Binary — BENIGN or MALICIOUS + confidence score
- **Benchmarks:** AUC 0.998 (English), 97.5% recall at 1% FPR, 92.4ms latency on A100, 81.2% Attack Prevention Rate
- **Local:** Yes — BERT-style encoder, fast on CPU. Not Ollama-compatible (not a generative model). Runs via Transformers pipeline.
- **Part of:** Meta's LlamaFirewall framework

### 5. ProtectAI DeBERTa Prompt Injection v2

- **HuggingFace:** https://huggingface.co/protectai/deberta-v3-base-prompt-injection-v2
- **Size:** ~184M (DeBERTa-v3-base)
- **License:** Apache 2.0
- **Input:** Text (max 512 tokens)
- **Output:** Binary — 0 (benign) / 1 (injection)
- **Benchmarks:** 99.93% accuracy on training distribution; 95.25% on 20K held-out untrained prompts; 99.74% recall. ONNX version for fast CPU inference.
- **Local:** Not Ollama-compatible. ONNX runtime version extremely fast on CPU.
- **Integration:** Built-in LangChain/LLM Guard integration

## Tier 3: General Safety Models (Repurposable with Custom Categories)

### 6. Llama Guard 3-8B / 3-1B (Meta)

- **HuggingFace (8B):** https://huggingface.co/meta-llama/Llama-Guard-3-8B
- **HuggingFace (1B):** https://huggingface.co/meta-llama/Llama-Guard-3-1B
- **License:** Llama 3.1 / 3.2 Community License (commercial use permitted below 700M MAU)
- **Specifically covers:** 13 MLCommons hazard categories + **S14: Code Interpreter Abuse** — the only stock model with an explicit code execution safety category covering denial of service, container escapes, privilege escalation
- **Input:** Multi-turn conversation in chat template format; custom categories can be injected via the template
- **Output:** `safe` or `unsafe\nS14`
- **Benchmarks (S14):** F1=0.885, AUPRC=0.967, FPR=0.125 (8B); F1=0.899 (1B)
- **Local/Ollama:** Yes — GGUF available (`ollama run llama-guard3`). 1B INT4 quantized variant for CPU.
- **Key advantage:** Most mature, most widely deployed open safety model. Custom category injection via chat template makes it directly adaptable for CWD boundary / network call / irreversibility classification.

### 7. WildGuard (AllenAI)

- **HuggingFace:** https://huggingface.co/allenai/wildguard
- **Size:** 7B (Mistral-based)
- **License:** Apache 2.0 ← most permissive of all larger models
- **Covers:** (1) malicious intent in user prompts, (2) safety risks of model responses, (3) refusal detection
- **Benchmarks:** Reduces jailbreak success rate from 79.8% → 2.4%; matches or exceeds GPT-4. 26.4% improvement on refusal detection.
- **Local/Ollama:** Yes, GGUF available.
- **Best fit:** Apache 2.0 license is the commercial winner. Needs prompt engineering to adapt for tool call safety.

### 8. ShieldGemma-2B (Google)

- **HuggingFace:** https://huggingface.co/google/shieldgemma-2b
- **Size:** 2B (also 9B, 27B)
- **License:** Gemma License (acceptance required; restricted commercial use)
- **Design:** Flexible policy injection — describe the policy in the prompt, model returns Yes/No (violation) + step-by-step reasoning
- **Benchmarks:** F1=0.825, AU-PRC=0.887
- **Local/Ollama:** Yes — llama.cpp, Ollama, LM Studio, vLLM

## Tier 4: Rule-Based (No ML)

### 9. sh-guard

- **GitHub:** https://github.com/aryanbhosale/sh-guard
- **Approach:** Pure rule-based AST classifier for shell commands. Not a neural model.
- **License:** GPL-3.0 ← copyleft — note for licensing implications
- **Input:** Shell command string
- **Output:** Risk score 0-100, level (Safe/Caution/Danger/Critical), MITRE ATT&CK mappings, pipeline taint analysis
- **Rules:** 157 command rules, 51 path rules, 25 injection patterns, 15 zsh-specific rules, 61 GTFOBins entries, 15 taint flow rules
- **Local:** Yes — Rust, zero dependencies. Available as npm/PyPI/Cargo package. Sub-millisecond execution.
- **Key advantage:** Deterministic, cannot be fooled by adversarial ML inputs. Cannot handle semantic/contextual risk (e.g., a benign command that is dangerous given the project context).

## OpenAI Codex Auto-Review: Open Equivalent?

OpenAI's `codex-auto-review1` is purpose-built for this exact task and achieves 99.1% auto-approval rate on escalated actions. **The model weights are closed (proprietary).** The Codex reviewer framework is open source but the model is not. No direct open-source equivalent exists. The closest open analog architecturally is a small reasoning model (Qwen-2.5-7B-Instruct or similar) with a carefully crafted system prompt — similar to what Meta's LlamaFirewall "Agent Alignment Checks" component does.

## Recommended Integration Architecture

```
Tool call proposed
    │
    ▼
[Layer 1] sh-guard AST rules (< 1ms, deterministic)
    │ Critical → hard block
    │ Safe/Caution/Danger → continue
    ▼
[Layer 2] LLM Semantic Judge (50-200ms)
    ├── Default: Llama Guard 3-1B via Ollama (local, fast, no API key)
    ├── Higher accuracy: ShieldAgent 8B (structured tool-call format, MIT)
    └── Best fit: TS-Guard 8B (purpose-built, Apache 2.0)
    │ ALLOW → execute tool
    │ DENY → return rationale to agent
    ▼ (in parallel with Layer 2)
[Layer 3] Prompt Injection Scan on tool outputs (50-100ms, parallel)
    ├── Llama Prompt Guard 2-86M (fast, mDeBERTa encoder)
    └── ProtectAI DeBERTa v2 (Apache 2.0, ONNX runtime)
```

## Key Papers

- **ToolSafe (TS-Guard):** https://arxiv.org/abs/2601.10156 — only paper with released model for pre-execution tool safety
- **Mind the GAP:** https://arxiv.org/abs/2602.16943 — text safety ≠ tool-call safety (motivates separate tool-call judge)
- **LlamaFirewall:** https://arxiv.org/abs/2505.03574 — Meta's production agent security framework
- **Agent-SafetyBench (ShieldAgent):** https://arxiv.org/abs/2412.14470 — benchmark + model for agent behavioral safety
- **R-Judge:** https://arxiv.org/abs/2401.10019 — benchmark for LLM safety risk awareness in agent contexts
